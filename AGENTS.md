# Codex 工作规则

## Git 版本管理

- 每次修改文件前，先检查 Git 状态。
- 不要覆盖用户已有改动；如果发现用户未提交的改动，先说明再继续。
- 修改完成后，先检查差异并给出简短摘要。
- 提交信息要简短清楚，说明这次做了什么。
- 只有用户明确说“提交”时，才创建 Git 提交。
- 只有用户明确说“推送”时，才推送到 GitHub。
- 大改动、新功能、风险较高的修改，先新建 `codex/` 开头的分支。
- 普通小改动可以在当前分支完成，但提交前仍要复查状态。

## 文件清理

- 执行批量删除文件任务时一定要获得用户同意。
- 删除前先说明将删除哪些内容，尽量选择可恢复或范围明确的处理方式。
